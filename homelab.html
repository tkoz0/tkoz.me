Homelab

This page describes my janky homelab setup. Someone has to pay the power bill for this thing... (around $100/month)

Everything is connected to a cheap 8 port gigabit switch which is connected to a router. I have everything on a wooden rack I designed and built myself. Some of the equipment is attached to an APC Smart UPS SUA1500RM2U.

My primary interest with homelabbing is high performance computing (HPC). It is also a good opportunity to learn to setup things myself and a much cheaper alternative to cloud computing. Used x86_64/amd64 based servers are probably the most power efficient option I have for heavy computing that is not prohibitively expensive.

Rimuru (main server)

Named after the main character in the anime Tensei Shitara Slime Datta Ken (Tensura). I use this one for data storage and running various services. This was previously used for computation before I bought more hard drives to move my data storage from Lacia, the server I used previously for data storage.

  • Dell PowerEdge R720xd
  • 1x Intel Xeon E5-2643 v2 (6C/12T 3.5GHz)
  • 64GB (2x32GB) PC3-10600R memory
  • 12x8TB SAS HDD in ZFS RAIDZ2
  • Dell PERC H310 (flashed to IT mode)
  • 1TB PCIe NVMe SSD for OS
  • 2x750W Power Supplies
  • Ubuntu Server 22.04
  • Used from July 2022 - Present

Lacia (compute server)

Named after the character Lacia in the anime Beatless. I use this one for running heavy computing projects. These include my own code, performance experiments, and other various dumb ideas. The large amount of storage and RAM make this nice to use for certain applications such as y-cruncher.

  • Dell PowerEdge R720
  • 2x Intel Xeon E5-2697 v2 (24C/48T 2.7GHz)
  • NVidia Tesla P100 16GB GPU
  • 512GB (16x32GB) PC3L-10600R memory
  • 8x4TB SATA HDD
  • Dell PERC H710 (flashed to IT mode)
  • 1TB PCIe NVMe SSD for OS
  • 2x750W Power Supplies
  • Ubuntu Server 22.04
  • Used from July 2020 - Present

Shirase (decommissioned)

Named after the main character from the anime Battle Programmer Shirase (BPS). One memory channel is bad. I found this in the trash at RPI and used it for a while before I got the R720xd (named Lacia). It used to be an extra server I used for testing software installation/setup to get things working before repeating it.

  • IBM System x3530 M4
  • 250GB SATA SSD
  • 2x2GB + 2x4GB PC3-10600E memory
    • One memory channel is bad
    • Usable RAM is 6GB
  • Intel Core i3-4170 (2C/4T 3.7GHz)
  • 2x460W Power Supplies
  • Used from 2020 - 2022

Kurumi (compute server)

Primarily named after Kurumi, the hacker character in the anime Lycoris Recoil. Also happens to be the same name as Kurumi from the anime Date A Live. This is another compute server, used for CPU bound code with low memory requirements since it has a lot less memory and storage than Lacia has.

  • Dell PowerEdge R630
  • 2x Intel Xeon E5-2698 v4 (40C/80T 2.2GHz)
  • 32GB (4x8GB) PC4-2400T memory
  • 500GB SATA SSD
  • Dell PERC H330 (not in IT mode yet)
  • 2x750W Power Supplies
  • Ubuntu Server 22.04
  • Used from July 2023 - Present
